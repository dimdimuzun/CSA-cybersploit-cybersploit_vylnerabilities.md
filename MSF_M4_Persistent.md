# Модуль 4. Постексплуатація та закріплення (Post-Exploitation/Persistence)

---

Після того, як ви отримали сесію (наприклад, через Meterpreter), ваша задача — дослідити систему, підвищити привілеї та гарантувати собі доступ після перезавантаження.

--- 

### 1. Розвідка всередині системи (Post-Exploitation Enumeration)
Перш ніж щось змінювати, потрібно зрозуміти, куди ви потрапили.

* Linux:
   
    whoami && id          # Хто я? (root чи звичайний юзер)
    uname -a              # Версія ядра (можливість Kernel Exploit)
    cat /etc/shadow       # Спроба прочитати хеші паролів (потрібен root)
    
* Windows:
   
    whoami /priv          # Перевірка привілеїв (шукаємо SeImpersonatePrivilege)
    net user              # Список користувачів
    systeminfo            # Версія ОС та встановлені патчі
    
---

### 2. Підвищення привілеїв (Privilege Escalation)
Якщо ви зайшли під обмеженим користувачем (наприклад, vagrant або www-data), ви не зможете встановити бекдор у системні папки.

* Приклад (Linux): Пошук файлів із SUID-бітом.
    * Метод: find / -perm -u=s -type f 2>/dev/null
    * Експлуатація: Якщо в списку є вразливий бінарний файл, можна отримати права root.
* Приклад (Windows): Експлуатація вразливостей сервісів або Token Impersonation.
    * Метод: У Meterpreter використати команду getsystem.
* Захід усунення: Принцип найменших привілеїв. Обмеження прав SUID та регулярне оновлення ядра/патчів безпеки.



---

### 3. Закріплення в системі (Persistence)
Це створення "чорного входу", який дозволить повернутися в систему навіть після перезавантаження або зміни пароля користувачем.

#### А) Створення нового адміністратора (Найпростіший метод)
* Windows: net user hacker Password123 /add та net localgroup administrators hacker /add.
* Захід усунення: Моніторинг логів створення нових користувачів та аудит групи Administrators.

#### Б) Використання запланованих завдань (Scheduled Tasks)
Це найбільш непомітний спосіб для Windows.
* Експлуатація: Створення завдання, яке щогодини запускає ваш реверс-шелл.
   
    use exploit/windows/local/persistence_schtasks
    set SESSION <ID>
    run
    
* Захід усунення: Регулярна перевірка Task Scheduler на наявність невідомих скриптів.

#### В) SSH-ключі (Linux)
* Експлуатація: Додавання вашого публічного ключа в файл /home/user/.ssh/authorized_keys. Тепер ви можете заходити без пароля.
* Захід усунення: Заборона SSH-входу за паролем (тільки ключі) та аудит файлів authorized_keys.

---

### 4. Вихід та приховування слідів (Clearing Tracks)
Етичний хакер (або зловмисник) має залишити систему в тому ж стані, або видалити докази своєї присутності.

* Очищення логів (Windows): В Meterpreter команда clearev.
* Очищення історії (Linux): history -c або видалення файлу .bash_history.
* Захід усунення: Використання віддалених серверів збору логів (SIEM), куди логі передаються в реальному часі. Навіть якщо хакер видалить їх локально, вони залишаться на сервері безпеки.

### Послідовність для VirtualBox:
1. Отримайте доступ через будь-який експлойт з минулого модуля.
2. Перевірте свої права (getuid).
3. Виконайте persistence модуль в MSF.
4. Перезавантажте віртуальну машину в VirtualBox.
5. Переконайтеся, що ваш handler автоматично отримав нову сесію після завантаження ОС.
6. Опишіть, як адміністратор міг би це помітити (наприклад, через перегляд списку процесів або автозавантаження).

Цей модуль завершує логічний ланцюжок атаки: Розвідка -> Доступ -> Закріплення -> Приховування.
