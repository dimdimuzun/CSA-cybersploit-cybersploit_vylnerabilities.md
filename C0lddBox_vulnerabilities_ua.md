Ось **перелік вразливостей та слабких місць**, з якими ви стикнулися під час проходження цього CTF, у логічній та навчальній формі (підходить для звіту):

---

## **Перелік виявлених вразливостей**

1. **Відсутність сегментації мережі та захисту від розвідки**

   * Цільова машина безперешкодно виявляється за допомогою `netdiscover`.
   * Відсутні механізми приховування або обмеження мережевого сканування.

2. **Відкриті та доступні сервіси без додаткового захисту**

   * Вебсервіс (HTTP) доступний без обмежень.
   * Відсутні механізми rate limiting або IDS/IPS на етапі початкового сканування (`nmap`).

3. **Розкриття структури вебзастосунку**

   * Можливість перерахування директорій і файлів за допомогою `gobuster`.
   * Відсутність захисту від directory brute-force (robots.txt, WAF, правила сервера).

4. **Використання WordPress без належного hardening**

   * Доступна стандартна сторінка логіну WordPress.
   * Відсутні обмеження на кількість спроб входу.

5. **Перерахування користувачів WordPress (User Enumeration)**

   * Можливість визначити імена користувачів через `wpscan`.
   * Це значно спрощує подальший брутфорс паролів.

6. **Слабкий пароль адміністратора**

   * Пароль користувача `c0ldd` присутній у словнику `rockyou.txt`.
   * Відсутня політика складності паролів та захист від brute-force атак.

7. **Можливість редагування PHP-коду через адмін-панель WordPress**

   * Користувач із високими правами може редагувати шаблони тем.
   * Відсутні обмеження на виконання довільного PHP-коду.

8. **Відсутність контролю виконання коду на сервері**

   * Успішне впровадження PHP reverse shell через `footer.php`.
   * Відсутні механізми SELinux / AppArmor або обмеження PHP-функцій.

9. **Зберігання облікових даних у відкритому вигляді**

   * Файл `wp-config.php` доступний для читання.
   * У ньому містяться облікові дані, які повторно використовуються для системного користувача.

10. **Повторне використання паролів (Password Reuse)**

    * Пароль з WordPress використовується також для доступу до системного користувача `c0ldd`.

11. **Неналежна конфігурація sudo (Misconfigured sudoers)**

    * Користувач `c0ldd` має право запускати небезпечні бінарні файли через `sudo`:

      * `chmod`
      * `vim`
      * `ftp`
    * Без обмежень (`NOPASSWD`, без drop privileges).

12. **Можливість підвищення привілеїв через GTFOBins**

    * Успішне отримання root-доступу через:

      * `sudo chmod` (зміна прав доступу)
      * `sudo vim` (shell escape)
      * `sudo ftp` (виконання shell)
    * Відсутні обмеження `secure_path`, `noexec`, `env_reset`.

13. **Зберігання чутливої інформації у закодованому, але не захищеному вигляді**

    * Файли з Base64-кодуванням створюють ілюзію захисту, але легко декодуються.
    * Base64 ≠ шифрування.

---

## **Загальний висновок**

CTF демонструє **ланцюжок типових помилок конфігурації**:
від слабкого вебзахисту та паролів → до критичної помилки в `sudoers`, що дозволяє повнісьюй зкомпрометувати системи.
